# Windows Changelog

## 1.7.0 — 2026-07-19

- 管理器窗口和系统托盘新增自动/手动更新检查，显示新版本或可安装在线主题数量。
- 复用内置 Node.js 验证 Ed25519 清单签名、HTTPS 地址、版本、大小与 SHA-256 后下载 NSIS 安装器。
- 更新辅助进程等待旧管理器退出，静默安装到新的版本化引擎目录并自动启动新版，Codex 保持运行。
- 新增官方在线主题同步脚本，严格检查 ZIP 路径和 schema 2 包后原子写入持久主题库。

## 1.6.1 — 2026-07-19

- 与 macOS 补丁版本同步发布；Windows 托盘常驻、实时状态和快捷切换行为保持不变。

## 1.6.0 — 2026-07-19

- 管理器常驻 Windows 系统托盘，关闭主窗口时隐藏到后台，双击托盘图标可恢复窗口。
- 托盘菜单显示当前主题和 Codex 实时连接状态，并可直接快速切换、恢复原版或退出。
- 增加命名互斥量与窗口唤醒事件，重复启动只会唤醒已有管理器，不会产生多个托盘实例。
- 后台每 3 秒同步主题库、活动主题与运行状态。
- 初音未来主题改为跟随系统自动切换浅色、深色调色板。

## 1.5.0 — 2026-07-19

- 内置 `codex-skin-theme-creator` Skill，通过 Codex 对话创建或重做主题后自动写入持久用户主题库。
- “主题接入”页新增 Skill 安装状态、重新安装和目录入口。
- Skill 直接复用 Windows `theme-package.ps1` 创建核心，保持 schema 2、图片裁切、内置 ID 保护和原子替换规则一致。
- 管理器监测主题库变化并自动刷新，新增主题无需重启管理器。
- NSIS 安装器自动部署并更新 Skill，发布版本统一为 `1.5.0`。

## 1.4.3 — 2026-07-18

- 主题切换复用现有注入监视器并立即热应用，不再因停止 Node 进程的 Windows 误报回退管理器选中项。
- 注入监视器会检测当前主题文件变化，自动重载并重新注入已连接的 Codex 页面。
- 运行状态改为校验 Codex 本机 CDP 端点，修复主题可切换但管理器显示“等待连接”的问题。
- 皮肤库末行加入创建和导入操作卡片，补齐三列布局。
- 修复“切换中...”按钮在 Windows 禁用态下文字颜色失效，并抑制 PowerShell CLIXML 进度输出。
- “蔚蓝档案 · 青春合影”聊天页使用适配五人合影的满高取景，最右侧人物不再被横向裁掉。
- 设置页侧栏、内容卡片、输入控件、下拉菜单和文字统一继承当前主题变量，浅色与深色主题都保持完整对比度。
- 注入器识别设置页和分类加载状态，停留在设置页也能一键热切换主题，宠物窗口保持显示。

## 1.4.2 — 2026-07-18

### 修复

- 所有 PowerShell 脚本使用 Windows PowerShell 5.1 可识别的 UTF-8 BOM，修复中文脚本被误解析为乱码后产生的引号和括号语法错误。
- 管理器、启动器和恢复入口通过 `wscript.exe` 隐藏启动，不再常驻 Windows Terminal 黑色进程窗口。
- 主题切换改用隐藏的异步子进程，分别处理实时成功、已保存待重启和真实失败，不再成功后弹出统一代码 1 错误。
- “Codex 默认原版”改为不关闭 Codex 的软恢复流程，停止注入器并实时移除皮肤；主题选择单独持久化，恢复原版后可直接再次切换。
- 引擎安装到 `engine-1.4.2` 独立目录，旧注入器运行时也不会占用新版本文件。

### 改进

- Windows 管理器重构为与 macOS 一致的深色侧栏、当前主题横幅、工具栏、搜索、主题卡片、主题接入页、运行状态页和卡片内一键切换布局。
- 管理器、切换器和创建/导入功能统一使用持久主题库 `%LOCALAPPDATA%\CodexDreamSkin\themes`。

## 1.4.1 — 2026-07-18

### 修复

- 支持在 Codex 正在运行时直接安装或升级，安装过程保持当前窗口开启，主题在首次应用时生效。
- 配置写入继续使用 UTF-8 原子事务，并对 Codex 同时写配置造成的短暂冲突自动重试。
- 其他安装异常会保存到 `%LOCALAPPDATA%\CodexDreamSkin\install-error.log`。

### 修复

- 所有内置主题和后续主题清单统一保持宠物层显示。
- 插件/技能库验证改查库页面容器，不再因该页面没有聊天输入框而误报失败。
- 芙宁娜和 Saber 主图以 `gpt-image-2` 高质量模式重新生成，强化官方角色特征、面部精度、服装细节、光影层次和 3:1 主题构图；聊天页加入各自的构图焦点与正文阅读层。
- 明日香主图重新生成，人物面部移至横向约 59% 并保留左侧第三新东京市场景；芙宁娜主图重新取景并减弱首屏与内容页的白色遮罩。
- Windows PowerShell 不再用空备份路径调用 `File.Replace`；配置原子替换改用同目录唯一临时备份，修复安装阶段的 `ArgumentException`。
- 月薪喵插件/技能页不再继承 Codex 暗色主表面令牌，搜索栏、安装按钮和提示浮层统一为浅色主题表面。
- CRLF 配置中的多行数组现在会在修改前正确识别并停止；无其他配置变化时恢复会精确还原原文件结尾，不再多留空行。
- 安装与 `-RestoreBaseTheme` 现在严格按 UTF-8 读取，保留原换行风格，并以无 BOM、同目录原子替换方式写回 `config.toml`，避免中文项目名称乱码或导致 Codex 无法启动。
- 遇到带 BOM/无 BOM 的 UTF-16、NUL 字符、无效 UTF-8 或写入期间被其他程序改动的配置时停止修改，不再静默转码或覆盖较新的内容。
- 安装会在当前注册包或 state 记录的旧 Codex 仍运行时明确提示先关闭；配置临时文件写完后会在原子替换前再次核对原始字节，进一步缩小并发覆盖窗口。
- 配置恢复只修改 `[desktop]` 内的外观键，不再误碰其他 section 的同名配置；新增 `-RecoverConfigBackup` 用于显式恢复安装前原始文件，并先保存当前文件。
- 完成配置恢复后会归档本轮安装前备份，使下一次安装重新保存当时的配置，避免重复安装使用过期主题值。
- schema 3 记录的旧 injector PID 只有在 Node 精确路径、脚本命令行、端口、Browser ID 和进程启动时间匹配时才会停止；兼容旧 state 时仍要求原 state 含脚本路径和端口，且 PID 仍匹配 `node.exe`、脚本与 watch 参数，无法确认便归档而不结束进程。
- 启动验证失败会停止 injector、清理状态，并把本次新开的 Codex 恢复为无调试口的普通启动。
- Restore 使用状态中记录的端口，先关闭运行态再写配置；失败时保留 state 并尽量正常重开 Codex，不再留下半完成状态或静默报告假成功。
- Store 更新后若旧版本仍持有已保存的 CDP，会按 state 中的精确路径关闭；检测到新旧版本同时运行时安全停止并提示人工处理。
- 支持带注释或引号的 `[desktop]` 表头与目标键；遇到转义同义键、多行字符串/数组、dotted key 或重复目标键时会在写入前明确停止，避免把合法但无法安全行编辑的 TOML 改坏。
- Store 更新后的旧路径只有在 Appx full name、family name、安装目录和可执行文件仍能与系统注册包匹配时才允许自动关闭；无法证明归属时保留状态并要求手动关闭。
- Store 更新时，仍在运行且身份有效的旧版本 CDP 会直接热重应用；旧版本若未开启 CDP，则在获得现有重启授权后关闭并启动当前注册版本，避免并行打开两个 Codex。
- 遇到 `[desktop.*]` 子表时会在写配置前停止，避免外观标量键与 TOML 子表冲突；热重应用验证失败时会尽力移除本次残余样式。
- Restore 不再要求当前环境仍能找到 Node；schema 3 清理会严格匹配安装时记录的 Node 路径，Node 已升级或卸载也不影响安全恢复。
- 插件或技能库页面现在也会被识别为有效 Codex 主界面，主题热切换不再停留在上一套样式。
- “OpenAI 是人民的 AI”移除首页顶部深色渐隐边，侧栏弱文字、悬浮菜单和会话提示条统一为浅色红白样式。

### 安全

- Codex 以 `--remote-debugging-address=127.0.0.1` 启动；同时校验监听 PID 对应精确的官方 Store 可执行文件。
- 说明：loopback 可阻止局域网访问，但 CDP 不验证同一 Windows 用户下的其他本地进程；不用皮肤时建议执行 Restore 关闭调试会话。
- Appx 发现要求 `SignatureKind=Store` 且不是 development mode，同名开发包或侧载包不会被当作官方 Codex 启动或关闭。
- injector 只连接相同端口、page ID 与路径一致的 loopback WebSocket，并在注入前确认真实 Codex shell DOM 标记。
- watcher 绑定启动时的 CDP Browser ID，并持续持有 Browser WebSocket 作为身份锚；原浏览器关闭或端口被复用时直接退出，不会连接到新端点。
- CDP HTTP、WebSocket 建连与命令均加入超时，HTTP 探测拒绝重定向，异常目标不会无限挂起或把探测带离 loopback。
- injector 日志与验证文件不再记录窗口标题、页面路由、页面文本或被拒绝 URL 的内容，只保留临时 target ID、结构标记和布局结果。
- 快捷方式不再静默携带 `-RestartExisting`；需要重启时先向用户确认。
- install、start、restore 和 verify 使用当前用户互斥锁，避免双击或并发命令竞争端口、配置和 state。

### 改进

- 与 macOS 共用十四套 schema 2 主题，固定为 `theme.json + background.png + preview.png` 单主图格式。
- 新增带主题预览的 WinForms 管理器、主题热切换脚本、应用图标和 NSIS 一键安装/卸载包。
- 安装包内置 Windows x64 Node.js 24 运行时，目标机器无需预装 Node。
- 默认端口被占用时自动在后续 100 个端口内选择空闲端口；显式指定的冲突端口仍安全失败。
- injector 会等待首轮注入完成再判定启动成功；目标异常时使用有上限的指数退避和限频日志，减少后台唤醒和日志膨胀。
- 明确要求 Node.js 22 或更新版本，并记录 `process.execPath`，兼容 PATH 中的启动转发程序。
- 带空格或结尾反斜杠的测试 profile 路径现在按 Windows 命令行规则引用。
